🌐 OPTIONS DE DÉPLOIEMENT EN LIGNE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔷 OPTION 1 : GITHUB PAGES (Gratuit, recommandé)
   
   Étape 1 : Créer un repository sur GitHub
      • Aller sur github.com
      • Cliquer "New repository"
      • Nom : "portfolio" ou "[votre-username].github.io"
   
   Étape 2 : Pousser le code
      cd /home/depinfo/PORTFOLIO
      git init
      git add .
      git commit -m "Initial commit - Portfolio professionnel"
      git remote add origin https://github.com/[username]/[repo].git
      git branch -M main
      git push -u origin main
   
   Étape 3 : Activer GitHub Pages
      • Aller dans Settings > Pages
      • Source : Deploy from branch
      • Branch : main / (root)
      • Cliquer "Save"
   
   ✅ Site disponible à : https://[username].github.io/[repo]


🔷 OPTION 2 : NETLIFY (Gratuit, très simple)
   
   Étape 1 : Créer un compte sur netlify.com
   
   Étape 2 : Drag & Drop
      • Glisser-déposer le dossier PORTFOLIO sur netlify.com
      • Attendre le déploiement (2-3 minutes)
   
   OU
   
   Étape 2 : Connexion GitHub
      • Connecter votre compte GitHub
      • Sélectionner le repository
      • Déploiement automatique à chaque commit
   
   ✅ Site disponible à : https://[nom-unique].netlify.app
   ✅ Possibilité d'ajouter un domaine personnalisé


🔷 OPTION 3 : VERCEL (Gratuit, rapide)
   
   Même principe que Netlify :
      • Créer un compte sur vercel.com
      • Importer le repository GitHub
      • Déploiement automatique
   
   ✅ Site disponible à : https://[nom].vercel.app


🔷 OPTION 4 : SERVEUR WEB CLASSIQUE
   
   Pour Apache ou Nginx :
      # Copier les fichiers
      sudo cp -r /home/depinfo/PORTFOLIO/* /var/www/html/
      
      # Donner les bonnes permissions
      sudo chown -R www-data:www-data /var/www/html/
      sudo chmod -R 755 /var/www/html/
