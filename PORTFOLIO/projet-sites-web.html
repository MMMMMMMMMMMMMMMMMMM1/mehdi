Création de sites web responsives

SAÉ S1 & S2 — Projets académiques | 2023 – 2024

HTML5 CSS3 JavaScript WordPress Responsive Design Accessibilité

Contexte du projet

Type de projet : SAÉ S1 & S2 — Situations d'Apprentissage et d'Évaluation
Cadre : Académique — Travaux individuels et en binôme
Durée : Plusieurs projets répartis sur deux semestres (2023 – 2024)
Objectif : Maîtriser les fondamentaux du développement web : structure HTML sémantique, mise en forme CSS, intégration JavaScript et gestion de CMS

Ces projets constituent mon premier contact avec le développement web professionnel. L'enjeu était d'aller au-delà du simple code qui "fonctionne" pour produire des interfaces accessibles, bien structurées et adaptées à tous les écrans.

Projets réalisés

Site vitrine WordPress (S1)

Création d'un site pour une association locale. Configuration du CMS, choix et personnalisation du thème, gestion des contenus et des plugins.

Portfolio HTML/CSS (S1)

Portfolio personnel entièrement codé à la main, sans framework. Découverte du Flexbox, des animations CSS et des bonnes pratiques sémantiques.

Site e-commerce responsive (S2)

Interface d'un site de vente avec catalogue produit, filtre par catégorie et panier. JavaScript pour les interactions dynamiques, design adaptatif mobile/desktop.

Contraintes techniques

  • Validation HTML W3C et CSS sans erreurs
  • Respect des normes d'accessibilité WCAG (attributs alt, structure de titres, contraste)
  • Design responsive : mobile-first avec media queries
  • Aucun framework JavaScript autorisé pour les projets S1 (JS vanilla uniquement)
  • Performance : temps de chargement optimisé (images compressées, CSS minifié)

Démarche de développement

Phase 1 — Maquettage & conception

  • Analyse des besoins utilisateur et définition de l'arborescence
  • Réalisation de wireframes sur papier puis sur Figma
  • Choix de la palette de couleurs, typographies et charte graphique

Phase 2 — Intégration HTML/CSS

  • Structure sémantique : header, nav, main, section, article, footer
  • Mise en page avec Flexbox et CSS Grid selon la maquette
  • Animations et transitions CSS pour l'interactivité visuelle

Phase 3 — JavaScript & dynamisme

  • Manipulation du DOM : affichage/masquage, filtres produits
  • Gestion du panier avec localStorage
  • Validation côté client des formulaires

Phase 4 — Tests & validation

  • Test sur plusieurs navigateurs (Chrome, Firefox, Safari)
  • Validation W3C HTML et CSS
  • Vérification de l'accessibilité avec des outils dédiés

Compétences acquises

HTML5 & CSS3

  • Structure sémantique et accessibilité (ARIA, alt, heading structure)
  • Layouts complexes avec Flexbox et CSS Grid
  • Animations CSS : keyframes, transitions, transform
  • Media queries et approche mobile-first
  • Variables CSS et organisation du code en composants

JavaScript

  • Sélection et manipulation du DOM (querySelector, addEventListener)
  • Gestion des événements utilisateur (click, input, submit)
  • Stockage local avec localStorage et sessionStorage
  • Fetch API pour les appels réseau
  • Validation de formulaires côté client

WordPress & CMS

  • Installation, configuration et personnalisation d'un CMS
  • Gestion des thèmes, plugins et widgets
  • Création de pages et articles avec l'éditeur Gutenberg
  • Gestion des droits utilisateurs et des médias

Méthode & outils

  • Maquettage sur Figma avant le développement
  • Utilisation de Git pour la gestion de version
  • Outils de debugging (DevTools Chrome)
  • Optimisation des performances web (Lighthouse)

Compétences humaines développées

  • Rigueur : Valider chaque étape avant de passer à la suivante, ne pas laisser de code "qui marche à peu près"
  • Souci du détail : Vérifier l'affichage sur différents appareils et navigateurs
  • Autonomie : Chercher la documentation MDN, Stack Overflow, apprendre de ses erreurs
  • Communication : Présenter ses choix de conception devant l'équipe pédagogique
  • Sens User-centric : Penser l'interface du point de vue de l'utilisateur final, pas du développeur

Ce que ces projets m'ont apporté

Ces SAÉ constituent le socle technique sur lequel tout le reste est construit. Sans maîtriser HTML/CSS/JS, rien de ce que j'ai réalisé ensuite n'aurait été possible — ni les interfaces de l'application d'enchères, ni ce portfolio lui-même.

Bilan personnel

  • Rapport au code : J'ai appris à ne pas copier-coller sans comprendre. Chaque propriété CSS que j'utilise aujourd'hui, je sais pourquoi je l'utilise.
  • Expérience utilisateur : Ces projets m'ont sensibilisé à l'accessibilité — ce n'est pas une option, c'est une responsabilité.
  • Première confrontation au client : Pour le projet WordPress, l'association avait de vraies attentes. Ça m'a appris à écouter, à reformuler, à livrer ce qui était demandé et pas ce qui me semblait "plus joli".

Point de fierté

Le site e-commerce du S2 fonctionne entièrement en JavaScript vanilla, sans jQuery ni aucune dépendance externe. Gérer le panier avec localStorage et afficher dynamiquement les produits filtrés m'a demandé de vraiment comprendre le DOM — et cette compréhension m'est utile à chaque projet depuis.
