SAÉ S3.B.01 — Projet académique en binôme | 2024

Java Spring Boot TCP Sockets Cryptographie RSA Scrum REST API

Contexte du projet

Type de projet : SAÉ S3.B.01 — Situation d'Apprentissage et d'Évaluation
Cadre : BUT Informatique — Équipe de 4 étudiants, méthode Agile Scrum
Durée : 5 sprints (semestre 3, 2024)
Objectif : Concevoir et développer un système client-serveur d'enchères à plis fermés sécurisé, en partant d'un besoin client volontairement flou

Le projet consistait à implémenter le protocole de Vickrey (enchères au second prix à plis fermés), un mécanisme d'enchère où chaque participant soumet son offre de façon confidentielle et chiffrée. Le vainqueur est celui qui offre le plus, mais ne paie que le montant de la deuxième offre la plus haute.

Contraintes techniques

  • Architecture client-serveur en Java avec sockets TCP bas niveau
  • Chiffrement asymétrique RSA des offres et signature numérique
  • Tiers de confiance (Autorité) pour l'ouverture sécurisée des plis
  • Interface web via Spring Boot avec endpoints REST
  • Gestion de projet Agile avec backlog, user stories et sprints définis

Architecture & Protocole

Architecture hybride

Cœur réseau — Sockets TCP Java

Protocole d'échange bas niveau entre Vendeur, Enchérisseurs et Autorité. Serveurs multithreadés pour gérer les connexions simultanées.

Interface web — Spring Boot

API REST pour l'inscription, l'authentification et le dashboard. Endpoints HTTP gérés par Spring Boot avec formulaires d'enchères.

Sécurité — Cryptographie RSA

Gestion des clés asymétriques publiques/privées, signature numérique des offres, calcul du vainqueur selon la règle du second prix.

Méthode — Agile Scrum

5 sprints avec backlog refinement, user stories, critères d'acceptation et relation client simulée.

Déroulement des sprints

Sprint 1 — Cadrage & Architecture

  • Analyse du besoin client, rédaction des user stories
  • Choix de l'architecture hybride TCP/Spring Boot
  • Setup du projet et des outils (Git, Maven)

Sprint 2-3 — Protocole réseau & Cryptographie

  • Implémentation des sockets TCP multithreadés
  • Chiffrement RSA et signature numérique des offres
  • Protocole de communication Vendeur/Enchérisseur/Autorité

Sprint 4 — Interface web & API REST

  • Création des endpoints Spring Boot (auth, dashboard, enchères)
  • Dashboard administrateur et formulaires
  • Intégration frontend/backend

Sprint 5 — Tests & Livraison

  • Tests d'intégration du système complet
  • Correction des bugs, stabilisation
  • Documentation et présentation client
